## 2. 核心冲突 (The Macro-Mismatch)

Kelvin 所在的市场环境与其资产组合之间存在一个 **"周期 & 流动性错配 (Cycle & Liquidity Mismatch)"**：

### 债券 / 收息资产问题
- **现状**: 2020 年低利率背景下，久期较长债券 & 收息资产看似稳健
- **问题**: 2025 年利率上升导致其市值大幅下跌 (浮亏 ~12–15%)，组合缺乏周期适应性

### 现金资产问题
- **现状**: 虽然流动性强，但被闲置在低息储蓄账户 (利率约 0.05%)
- **问题**: 在通胀与生活成本上升环境下不断被"隐性蚕食" (购买力下降)

> **结果**: 原本看似稳健、稳防守的配置，反而导致财富实际缩水 —— 无论是市值 (债券浮亏) 还是购买力 (现金被通胀 / 机会成本侵蚀)。

## 3. AI-Agent ("Wealth Compass") 的定位 & 推荐逻辑

### 定位原则
- 不主张激进投机 / 高风险产品
- 强调稳健、防守、流动性、稳固收益，并兼顾心理舒适度与合规性

### 推荐逻辑

1. **宏观归因 (Macro Attribution)**  
   帮助客户理解亏损 / 现金拖累的根源是宏观环境 (利率 / 通胀 / 流动性周期) 的变化，而不是个人判断失误。

2. **现金激活 (Cash Activation)**  
   将长期闲置现金 (Idle Cash) 作为优化标的。

3. **低摩擦、流动性高、可逆 (Low-Friction & Reversible)**  
   主推短期 / 短债 / 货币基金 / 短期国债 (Short-Duration Bond / T-Bill / Money Market Fund)；避免一次性 "重构组合 / 高风险 / 高波动" 投资。

4. **渐进式、试水 (Pilot / Phased Approach)**  
   先以小额 (e.g. SGD 200k) "试水"，观察表现，再逐步按需扩大。

5. **合规 + Advisor (RM) Gatekeeper 模式 (Digital Handshake)**  
   即使客户执行 "一键优化 (One-Click Optimize)" 操作，仍有人工 RM / Advisor 最后审核 / 批准，以满足银行合规要求 (尤其是大额资金流动)。

## 6. 成功标准 (KPI / Acceptance Criteria)

- **现金利用率 (Cash Utilization Rate)**: 从高于 30–40% (Idle Cash) 降至低于 10%–15% (流动现金占比显著下降)
- **组合年化收益率 (Portfolio Yield)**: 提升：从 ~1.2% → ≥ 2.5–3.0%
- **流动性固定收益 (Liquid Fixed Income) 占比**: 达到总资产 (AUM) 的 20%–30%
- **客户体验 / 满意度 / 信任度**: (对 AI-Agent + RM / Advisor) 显著上升 — 客户对下一步 "债券修复 (Bond Repair)" 建议持开放态度
- **合规通过率 / 操作失败率**: 极低 (例如 RM 审核通过 / 交易成功率达 100%，无重大客户 / 合规投诉)

## 7. 潜在 Objections / 风险 & 对应缓解 (Risks & Mitigations)

### 风险一: 客户心理排斥
- **风险**: 客户仍然心理排斥卖亏损资产 —— 即使短债 / 现金优化成功，也可能拒绝后续 "债券修复 / 重构组合" 建议
- **缓解**: 先强调 "现金优化 + 流动性 + 稳健收益" 的好处，不急于触及亏损资产；将 "债券修复" 作为后续建议，由 RM 以 "Market-Condition Review" 方式发起

### 风险二: 收益率波动
- **风险**: 短债 / 货币基金 / T-Bills 收益可能随利率 / 市场波动下降 —— 若未来利率转向，收益率可能下降 (或低于预期)
- **缓解**: 系统定期 radar-check，一旦收益下降或通胀 / 利率环境变化，主动提醒客户是否需要调整 / 赎回 / 再平衡

### 风险三: 合规风控顾虑
- **风险**: 合规 / 风控顾虑：AI 自动执行大额资金调动是否合规 / 是否被 RM / Compliance 拒绝
- **缓解**: 通过 "Digital Handshake + RM 最终复核" 机制，引入人工审批 (复核) 节点，确保满足银行 / 法规对大额资金流动的审批标准；流程设计透明、可审计、符合合规要求

### 风险四: 客户信任度不足
- **风险**: 客户对 "AI + 数字化交易 + 自动化建议" 信任度不足
- **缓解**: 强调 "Hybrid 模型 (AI 建议 + 人工 RM 审核)"；首次建议从小额度 (SGD 200k–500k) 开始试水，降低心理 / 信任门槛；并通过后续定期报告 + 人工 follow-up 建立信任

这个用户故事展示了 AI-Agent 在财富管理中的核心价值：

- **宏观视角** - 帮助客户理解市场周期变化
